# 项目规则

## C++ 源码是绝对真理

在此仓库中工作时：

- 将 `cpp/` 下的原始 C++ 实现视为行为和语义的绝对真理。
- Python、绑定层、测试以及周边胶水代码中的任何逻辑修复，都必须有对应的 C++ 源码依据。
- 在实施逻辑修复前，先定位能够确立预期行为的对应 C++ 文件、类、函数或控制流行为。
- 当修改影响逻辑或语义时，在分析、调试和代码审查说明中明确写出对应的 C++ 依据。
- 如果找不到对应的 C++ 依据，不要将修改表述为“逻辑修复”；应明确标注为假设、实验，或项目特定的偏离。
- 当 Python 行为与 C++ 不一致时，除非项目文档明确说明这是有意偏离，否则优先让 Python 向 C++ 对齐。
- 编写 Python 代码时，在可行的情况下尽量使用类型注解，尤其是函数参数、返回值和重要的局部抽象。
